# Название проекта

Пример реализации конвейера обработки HTTP-запросов с использованием middleware в ASP.NET Core.

---

## Запуск проекта

1. Открой `solution` в **Visual Studio 2022**.
2. Нажми **Run** (или `F5`).
3. Откроется браузер с **Swagger** — можно тестировать эндпоинты.

---

## Архитектура

Проект построен как **конвейер обработки HTTP-запросов**. Каждый middleware выполняет **одну функцию**:

### 🛠 Middleware

- **`LoggingMiddleware`** — логирует входящие запросы.
- **`TimingMiddleware`** — измеряет время выполнения запроса.
- **`ExceptionMiddleware`** — перехватывает исключения и преобразует их в JSON-ошибки.

### Контроллеры

Контроллеры содержат **только бизнес-логику**. Вся вспомогательная функциональность вынесена в middleware.

### Хранилище

Данные хранятся **в памяти**, реализовано через **singleton-сервис**.

---

## Почему такой подход?

Такая архитектура делает систему:

- **Расширяемой** — легко добавлять новые middleware.
- **Понятной** — каждый компонент отвечает за свою задачу.
- **Тестируемой** — бизнес-логика изолирована от инфраструктуры.

---

## Используемые технологии

- ASP.NET Core
- Swagger / OpenAPI
- In-memory хранилище
